Leetcode0001 两数之和

题目描述：给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。

  • 方法1：两次循环，最简单，效率也最低

enumerate函数的使用：enumerate() 函数用于将一个可遍历的数据对象(如列表、元组或字符串)组合为一个索引序列，同时列出数据和数据下标
四个数字的默认测试集得到的实际 runtime: 40 ms 预计是：20 ms

python for循环的若干总结：
1、for循环可以遍历任何序列的项目，如一个列表或者一个字符串。

2、执行循环的遍历方式是通过索引

结果：超出时间限制

  • 方法2：字典模拟Hash

此方法在方法一的基础上改进了，只需要一层循环即可找出答案。即根据当前遍历得到的元素index，查找target-index是否在剩余数组里出现，如果找得到，则返回其下标值；反之则说明没有该答案。

ElasticSearch配置

最近在给大boss做个查询系统，都是基于文本的，所以选择了ElasticSearch作为存储和搜索。Win10上的配置和基本使用有时间会整理一下，这里记录一下今天处理的小问题：安装analysis-ik插件。ES的更新很快，版本与版本之间的差距也很大，所以这里以相对较新的7.2版本为例：

下载analysis-ik源文件

github上下载对应版本的源文件（具体版本可参照readme）

打包

下载完成之后进入下载目录，解压缩，
修改pom.xml，以ES7.2为例，将ES版本改成7.2.0
默认好像是7.0.0，如果不改，ES将会启动不了，Kibana也用不了，一直显示disconnect error
cd到具体文件夹内
mvn clean package
等待完成后，BUILD SUCCESS

复制与重启

将target–>releases中的压缩包复制到ES目录的plugins中新建的analysis-ik文件夹下，解压缩

进入win10服务，找到ES，右键重新启动

多刷新几次，看到正在运行没退出的话，才算成功

使用

在mapping中配置analyser，粒度可选，具体可参考readme文件，后期也会整理具体使用方法
